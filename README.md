# 自律オーディション型 note量産システム 仕様書（最終確定版）

## 1. システム概要
本システムは、ユーザーがチャット欄に直接入力した雑多なメモや抽象的なテーマ（インプット）に対し、事前にストックされた最大50通りの良質な記事パターン（ベンチマーク）からAIが自律的に最適なスタイルを2つ厳選し、「2通りの構成案」と「2通りの本文初稿（各デフォルト2,500文字）」を、人間の手を煩わせることなくチャット欄に同時に一括出力・提案する、VS Code環境に最適化された成長型執筆システムである。

## 2. システムのデータフロー図
```text
【人間のアクション】                  【システム・AIの自律処理】
================================================================================

 [ネタをチャットに直接入力]
          │
          ▼
 [プロンプトテンプレート適用]
          │
          ▼
   【AIエージェント起動】 ───────► [benchmarks/ フォルダをスキャン]
                                           │ (50個のファイルを読み込み)
                                           ▼
                                  [ステップ1: オーディション]
                                           │ (最適なアプローチを2つ厳選)
                                           ▼
                                  [ステップ2: 2通りの構成案作成]
                                           │ (導入・WHY・HOW・FUTURE)
                                           ▼
                                  [ステップ3: 2パターンの本文同時執筆]
                                           │ (各デフォルト2,500文字)
                                           ▼
 < チャット欄に一括出力！ > ◄──────────────┘
          │
          ▼
 [人間が2つの初稿を読み比べる]
          │
          ├──► 【パターンAを採用】 ──► 細部を微調整してnoteへ投稿！🎉
          └──► 【パターンBを採用】 ──► 細部を微調整してnoteへ投稿！🎉
```

## 3. フォルダ・ファイル構成（アーキテクチャ）
```text
note-generator/
│
├── .cursorrules               # システム全体の共通思想・基本トーンの定義
│
├── benchmarks/                # 真似したい神記事（50パターン）の格納ディレクトリ
│   ├── README.md              # フォルダの運用ルール、累積ナレッジベース
│   ├── pattern_01.md          # 参考記事パターン01（以下、50まで作成済み）
│   │   ...
│   └── pattern_50.md
│
└── templates/                 # エージェントへの指示用プロンプトテンプレート
    └── auto_audition_writer.md # チャット入力対応のオーディション＆同時執筆プロンプト
```

## 4. 各ファイルの要件・詳細仕様
### 4.1. .cursorrules（共通統治ルール）
- **アイデンティティ：** 知性的、オープン、ロジカル、かつ熱量の高いテック・スタートアップの編集者・ライター。
- **禁止表現：** 「最高」「究極」などの客観性のない誇張表現は一切禁止。
- **レイアウト：** スマホでの可読性を最優先し、1文を短く保つ。ボールドや箇条書きをバランスよく配置。

### 4.2. benchmarks/（50パターンの引き出しと累積学習）
- **初期状態：** `pattern_01.md` 〜 `pattern_50.md` はプレースホルダーとして生成済み。
- **運用の思想：** 人間が良質な記事をストックし、AIが将来的に自律分析してジャンルラベルを割り振る。

### 4.3. templates/auto_audition_writer.md（自律オーディション・プロンプト）
- **ステップ1：** 50パターンから最適なものを2つ厳選（異なる切り口）。
- **ステップ2：** 共通フレーム（導入・WHY・HOW・FUTURE）を適用した構成案生成。
- **ステップ3：** 本文初稿（デフォルト各2,500文字）をチャット欄に出力。
