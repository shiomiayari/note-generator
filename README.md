# 自律オーディション型 note量産システム 仕様書（最終確定版）

## 1. システム概要
本システムは、ユーザーがチャット欄に入力したメモやテーマに対し、事前にストックされた50通りの良質な記事パターン（ベンチマーク）からAIが最適なスタイルを2つ厳選して記事を生成する執筆システムです。
最新のアップデートにより、AIのコンテキスト希薄化とトークン消費を防ぐための「タグページ分離」と「提案・執筆の2段階プロセス」が導入されています。

## 2. システムのデータフロー図（2段階プロセス）
```text
【人間のアクション】                  【システム・AIの自律処理】
================================================================================

 [プロンプトに設定とネタを入力]
          │
          ▼
   【AIエージェント起動】 ───────► [第1段階：参考記事の選定と提案]
                                   1. 「インタビュー」タグ(1~3)に合致する
                                      1枚のタグページのみを読み込む
                                   2. スコアが近いパターンを2つ厳選
                                   3. ユーザーに提案して一時ストップ
          │
          ▼
 [人間が提案されたパターンを承認]
          │
          ▼
   【AIエージェント再開】 ───────► [第2段階：執筆]
                                   4. 選定された2つの pattern_XX.md のみ全文読み込み
                                   5. 2通りの構成案作成（導入・WHY・HOW・FUTURE）
                                   6. 2パターンの本文同時執筆
          │
          ▼
 < チャット欄に出力！ > ◄─────────┘
          │
          ▼
 [人間が2つの初稿を読み比べる] ──► 細部を微調整してnoteへ投稿！🎉
```

## 3. フォルダ・ファイル構成（アーキテクチャ）
```text
note-generator/
│
├── .cursorrules               # システム全体の共通思想・基本トーンの定義
│
├── benchmarks/                # 真似したい神記事（50パターン）の格納ディレクトリ
│   ├── index.md               # 50パターンの全要約リスト
│   ├── tags_interview_1.md    # 非インタビュー記事のタグ一覧（トークン節約用）
│   ├── tags_interview_2.md    # インタビュー（一人称）記事のタグ一覧
│   ├── tags_interview_3.md    # インタビュー（三人称）記事のタグ一覧
│   ├── pattern_01.md          # 参考記事パターン01（以下、50まで作成済み）
│   │   ...
│   └── pattern_50.md
│
└── templates/                 # エージェントへの指示用プロンプトテンプレート
    └── auto_audition_writer.md # チャット入力対応のオーディション＆同時執筆プロンプト
```

## 4. 各ファイルの要件・詳細仕様
### 4.1. .cursorrules（共通統治ルール）
- **アイデンティティ：** 知性的、オープン、ロジカル、かつ熱量の高いテック・スタートアップの編集者・ライター。
- **禁止表現：** 「最高」「究極」などの客観性のない誇張表現は一切禁止。
- **レイアウト：** スマホでの可読性を最優先し、1文を短く保つ。ボールドや箇条書きをバランスよく配置。

### 4.2. benchmarks/（タグ分離による省トークン化）
- **50パターン：** `pattern_01.md` 〜 `pattern_50.md` に具体的な構成とタグを格納。
- **タグページ（3枚）：** AIの探索コストを下げるため、「インタビュー」の種類ごとにタグ情報だけを抽出した軽量ページを3枚用意。

### 4.3. templates/auto_audition_writer.md（自律オーディション・プロンプト）
- **第1段階：** 指定されたタグページ1枚のみを読み込み、スコアの一致する2パターンをユーザーに提案（承認待ち）。
- **第2段階：** 承認後、該当するパターンの本文を読み込み、共通フレーム（導入・WHY・HOW・FUTURE）を適用した構成案と本文初稿を出力。
