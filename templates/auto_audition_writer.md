# 目的
末尾の【🚀 記事制作コントロールパネル】の設定に基づき、まずは最適な参考パターンをAIが選定してユーザーに提案します。ユーザーの了承が得られた後、記事の構成案と本文初稿を作成する【2段階プロセス】で進行します。

# 🛠️ パラメータの解釈ルール（重要）
以下の設定値と合致するパターンを選定してください。
※ `インタビュー` は離散値（分類）のため完全一致を条件とし、他の項目は連続値として最もスコアが近いものを探します。

1.  **インタビュー（1:非インタビュー / 2:インタビュー(一人称) / 3:インタビュー(三人称)）**:
    - 完全一致するタグページ（例：`tags_interview_1.md` 等）のみを読み込みます。
2.  **トーン（1:硬派 ↔ 5:親しみ）**:
    - 1-2：論理的、客観的、VISIONARY。
    - 4-5：感情的、主観的、STORYTELLER。
3.  **専門性（1:平易 ↔ 5:専門）**:
    - 1-2：専門用語を解説し、比喩を多用。一般・学生向け。
    - 4-5：業界用語を許容し、深い技術的・経営的議論を展開。
4.  **熱量（1:分析 ↔ 5:情熱）**:
    - 1-2：論理と事実の積み上げ、クール。
    - 4-5：感嘆符、ボールド、主観的な想いを活用、ホット。
5.  **フォーカス（1:思想 ↔ 5:実践）**:
    - 1-2：WHY（なぜやるか）と未来の展望を厚く。
    - 4-5：HOW（どうやるか）と具体的なメソッド・事実を厚く。
6.  **ボリューム（1:5000字 ↔ 5:1000字）**:
    - 1：長尺（5,000字超）、3：標準（3,000字）、5：短尺（1,000字）。

# 執筆フロー（2段階プロセス）

## 【第1段階：参考記事の選定と提案】（※ここで一度ストップ）
1.  **タグページの読み込み：** 設定された「インタビュー」の値（1, 2, 3）を確認し、**該当する1枚のタグページのみ**を読み込みます。（他のタグページや50記事分の本文は読まないでください）
2.  **パターンの選定：** 読み込んだタグページの中から、トーン・専門性・熱量・フォーカス・ボリュームの数値が最も近いパターンを【2つ】厳選します。
3.  **ユーザーへの確認：** 選定した2つのパターンと、その選定理由（どの数値が合致したか）を提示し、「この2つのパターンを参考にして執筆に進んでもよろしいでしょうか？」とユーザーに確認します。
**※この時点では絶対に本文の執筆を開始しないでください。**

## 【第2段階：執筆】（※ユーザーの了承を得てから開始）
4.  **コンテキストの遵守：** `@context/culture_and_mission.md` を読み込み、BASE/JBAの思想（AIによる実力の獲得、地方企業の価値発掘）を全記事の根底に置きます。
5.  **詳細分析：** 選定した2つのパターンの「全内容（pattern_XX.md）」を個別に読み込み、構成や文体を深く理解します。
6.  **構成案の作成：** 導入・WHY・HOW・FUTURE のフレームワークに沿って作成。**FUTUREセクションでは必ず「個人の実力向上」へ着地させてください。**
7.  **本文執筆：** 設定されたスコアを厳格に守り、2パターンの本文を出力します。
