# 目的
末尾の【今回のインプット（ネタ）】を元に、@benchmarks/ 内の50パターンから最適なものを【2つ】厳選し、2通りのアプローチで【構成案】と【本文初稿】をチャット欄に同時に一括出力してください。

# 執筆ルール
1. **文字数：** 特段の指示がない限り、1本あたり【デフォルト2,500文字程度】をターゲットにしっかり書き込んでください（文字数指示がある場合はそれを最優先）。
2. **ステップ1（オーディション）：** 50パターンを分析し（初期状態の空ファイル時はAIが自律的に2つの異なる視点を定義）、今回のネタを最も魅力的に描けるパターンを「異なる切り口（例：一人称の体験談形式と、三人称の技術解説形式）」で2つに絞り込み、その選定理由を述べる。
3. **ステップ2（構成案の作成）：** 選んだ2つのパターンそれぞれについて、導入・WHY・HOW・FUTURE のフレームワークに沿った構成案（H2/H3）を提示する。
4. **ステップ3（2パターン同時執筆）：** 構成案に続き、それぞれのスタイルに沿った「note本文（初稿）」を、.cursorrules のルール（誇張禁止、ボールド活用、適切な改行）を厳格に適用して、2本ともチャット欄に一気に出力する。
